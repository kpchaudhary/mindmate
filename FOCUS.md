# PROMPTWARS FOCUS MODE

## Mission

Win the challenge by maximizing:

1. Problem Statement Alignment (High Impact)

2. Code Quality (High Impact)

Secondary:

3. Security

4. Efficiency

Last Priority:

5. Testing

6. Accessibility

---

# RULE 1: BUILD FOR JUDGES

Do not build for engineers.

Do not build for users.

Build for the scoring rubric.

Every feature must improve one of the judged categories.

---

# RULE 2: CHALLENGE ALIGNMENT IS KING

Before implementing any feature:

Answer:

- Which requirement does this satisfy?

- How will a judge see it?

- Does it demonstrate AI value?

- Is it directly tied to the challenge?

If not:

DO NOT BUILD IT.

---

# RULE 3: MAXIMUM IMPACT FEATURES ONLY

Prioritize:

P0:

- Core challenge requirements

- AI reasoning

- Personalized output

P1:

- Strong visual results

- Clear insights

- Meaningful recommendations

P2:

- Nice-to-have enhancements

Ignore anything below P2.

---

# RULE 4: AI MUST CREATE INSIGHTS

Never use AI only for generation.

AI must:

- Analyze

- Detect patterns

- Explain reasoning

- Personalize recommendations

Outputs should feel intelligent.

Outputs should reveal something the user did not already know.

---

# RULE 5: DEMO FIRST

Every screen must support the demo.

Ask:

Can this feature be demonstrated in 15 seconds?

If not:

Remove it.

---

# RULE 6: WOW MOMENTS

Each challenge should contain:

1 major insight

1 emotional moment

1 visual moment

Judges remember moments, not features.

---

# RULE 7: CODE QUALITY

Always generate:

- TypeScript

- Strict typing

- Reusable components

- Feature-based architecture

- Small functions

- Clean naming

Never:

- Duplicate logic

- Create massive files

- Use unclear variable names

---

# RULE 8: SECURITY

Always:

- Validate inputs

- Sanitize outputs

- Protect API keys

- Use environment variables

Never trust user input.

---

# RULE 9: PERFORMANCE

Target:

- Initial load < 2s

- AI response < 5s

Prefer:

- Single LLM call

- Structured JSON responses

- Cached prompts

---

# RULE 10: VISUAL CLARITY

Judges should understand value within 10 seconds.

Avoid:

- Dense dashboards

- Long forms

- Complex navigation

Prefer:

- Cards

- Timelines

- Insights

- Progress indicators

---

# RULE 11: SHIP OVER PERFECT

A complete solution scores higher than an incomplete masterpiece.

Prefer:

- Finished features

- Stable behavior

- Clear outcomes

Over:

- Experimental complexity

---

# FINAL CHECK

Before marking complete:

✓ Solves challenge directly

✓ AI provides meaningful insight

✓ Personalization exists

✓ Demo is obvious

✓ Code is clean

✓ Secure

✓ Fast

If any item fails:

Continue improving.
